## Overview

The process for submitting chapters for review, collecting feedback, and incorporating revisions.

## Submission for Review

1. Complete a draft chapter in `30-Development/Drafts/` or `40-Manuscript/Chapters/`
2. Tag the chapter with `#stage/review`
3. Note the submission in [[Editorial-Calendar]]
4. Assign reviewers in [[Review-Assignments]]

## Review Period

[Standard turnaround time for reviews — e.g., 2 weeks for developmental, 1 week for line edits]

## Feedback Collection

1. Reviewers provide feedback using `#pm/feedback` tag
2. Feedback is compiled in [[Feedback-Summary]]
3. Author reviews all feedback before making changes

## Revision

1. Author addresses feedback and updates the chapter
2. Document changes in the chapter's Change Log
3. Update stage tag from `#stage/review` to `#stage/revision` or `#stage/editing`
4. If significant structural changes, re-submit for review

## Approval

1. When the editor approves, update tag to `#stage/final` or `#ms/final`
2. Record approval in `00-Project-Management/Decisions/Decision-Log.md`
